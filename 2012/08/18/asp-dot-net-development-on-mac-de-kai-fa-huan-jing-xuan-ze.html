ASP.net development on mac 的開發環境選擇

Aug 18, 2012

要開始在Mac上面開始開發ASP.net的web application了，這一兩天嘗試著裝，從本來想要Mac直上ASP.net一直退讓到裝上虛擬機器才順利。

Mono + Monodevelop

一開始本來就是從Mono開始，想說過了很多年的promoting應該可以變得很完整很順利，結果雖然裝是裝好了，Toolbox卻一直是empty的狀態！！！！！去search很多forum以後發現有人也有這問題，但是全都無解以後我就決定不要浪費時間來找這個東西，嘗試了幾小時以後就放棄了，然後決定使用虛擬機器來裝上windows 7直接跑visual studio系列。

虛擬機器 - Virtualbox

是Sum下面的免費產品，但是也因為免費，裝的過程很ㄎㄟˊ，windows 7打開來的時候都無法安裝然後一直跳出錯誤，幾小時以後也是崩潰的放棄。

虛擬機器 - VMware fusion 4

這就是會charge的mac系虛擬機，但是真的是一分錢一分貨，節省很多時間，輕輕鬆鬆安裝完成。接下來就是要裝上visual studio。選版本的時候著實令我苦惱好一陣子，因為現在被限定要選擇 .Net Framework 3.5 所以不能裝太新的版本，然後一時之間我就先裝上了Visual studio 2010 Express，然而畢竟是Express版，所以裡面真的很少東西，也無法選擇 .Net Framework 的版本，所以就打叉。然後無聊的時候又裝了Visual studio 2012 Express，畫面還真有質感，但是只支援 .Net Framework 4.0 以上，所以沒有辦法，只好花了很長的時間裝上了Visual Studio 2010 professional，真的超久(崩潰…)，現在就可以好好的使用Mac來開發ASP.net了。

–

後記

只能說ASP.net還是要在windows 7上面開發才最順手，畢竟是Microsoft的主力，真的沒有別的環境更適合了，就算是順利地把Mono勉強裝起來，也只是ASP.net的低等公民，這是沒有辦法的事情。但是其他的語言，Mac親和太多了，所以我不可能會放棄Mac，光是內建java, python等等等的就狂勝一大截，還有說不完的好處，所以事情就是這樣。起碼最後我不用切割成固定的雙系統，要藉由不斷重開機來交替切換OS這一點已經很感謝，然後三根手指滑來滑去就可以輕輕鬆鬆的回到適應的OS X，算是很OK啦。
