開工日：年度閱讀書籍

Feb 18, 2013

休了九天年假，把所有的時間都花給陪伴家人，看了很多本來買來一直沒時間翻的書，還去了很多漂亮的地方，感覺又回復動力開開心心的開工了。

軟書

  • Rework(工作大解放)

37 singles 品質保證。 其實這本已經不止看了一遍了，有趣的是，我覺得隔一陣子再看總會有新感受，另外就是可以回想審視自己還有哪些壞習慣和壞觀念。

  • 軟體構築美學：當專案團隊遇上失控程式，最真實的解決方案 (Brownfield Application Development in .Net)

自從工作上碰到無數的棕地程式之後，我就開始把注意力放在測試框架和design pattern上面，這本書以情境帶入解決方案，而且對於實務上會發生的各種情況講的很到位，案子不可能永遠都是全新開發的專案，以我而言，接手old code是更常有的情況，大力推荐這本書，但是能不能順利導入裡面的各種做法就是要面對團隊文化的問題了，這才是最艱困的麻煩。

  • 笑談軟體工程－敏捷開發法的逆襲－導入 Scrum，讓你的軟體開發人生從黑白變彩色！

剛好在參加2013 WebConf的時候在天瓏看到就順手帶了一本回家，這也是源於我對接手別人code的不滿所致，看來我目前最迫切的願望就是可以把垃圾都清乾淨，不過目前也只看到前面而已，Teddy的blog是一直都有在看的，希望這本書能帶來更多的一些什麼。

  • 快思慢想(Thinking, Fast and Slow)

主要是在講人的直覺是不可靠的（我知道，因為我的直覺對於生活上的一切都很不可靠例如樂透XD），其實是關於行為經濟學，裡面提供實驗和解說。生活上一直不停地在做選擇，我一直知道這件事（雖然書裡說人是無意識的一直在選擇），可是可能由於我一直有意識到「選擇」，也意識到「直覺不可靠」，所以每當面臨選擇我會產生壓力，然後最後就乾脆隨便決定，下決定前的思考方式非常的重要，所謂的快思就是直覺式思考，慢想就是邏輯式思考。人其實需要足夠的資訊才能做出更正確的決斷，何時該用哪一種，或是如何兩種配合，使自己的決策思考能夠更快也更合理。

硬書

  • Design Patterns

Gof的版本，應該不需要特別介紹。真的很難啊啊… 可是卻讓我想到一件事情，難道design patterns是專屬於物件導向語言的概念嗎？是不是像C language這種函數導向的語言就沒有design patterns可言呢？

  • SQL Server 2012 SSIS 整合服務

雖然我不是DBA，但是實務上卻不斷地碰到Database排程的問題，乾脆自己把SSIS學起來好了，因為Database的知識和技術對我目前來說真的是苦手中，更不用說NoSQL等等分散式的，還有SQL server以外的SQL是否有類SSIS的工具可學，大概要從這個大洞開始補了… SSIS我覺得極為不具彈性（和難以理解其邏輯），但是為了要接手修補過去失效的jobs，只好先硬吃起來，希望有朋友可以推薦更好的領域。
