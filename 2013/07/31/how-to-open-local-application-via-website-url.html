[How To] 怎麼讓網頁連結可以呼叫本機程式

Jul 31, 2013

其實事情的來由是這樣，我要做一個 google maps 應用，並且我希望，當我點擊地圖上的標記點，就可以打開我本機裝的軟體同時帶入相關資訊作為參數。

會有這個需求是因為地圖上的標記點要開啓的程式繁多且各不相同，如果是寫成本機軟體當然不成問題，然而一旦希望在軟體界面上使用 google maps，就必須要自幹一個瀏覽器框在軟體上面，然後這個地圖上的標記點又必須每秒刷新，可想而知performance會有多慘，於是寫成 Web application 勢在必行。

但這麼做之後，就是要通過 Url 來叫起那些各不相同本機程式了。

仔細想想，好像其實做了這件事的軟體也不少，例如逛 iTunes Store，”View In iTunes” 總是會把你電腦中灌的 iTunes 叫起來，還有檔案續傳軟體像是迅雷，以及像是搜狐 client 端，快播等等，都是在點擊觀看以後會把本機的程式叫起來，所以看起來這件事完全是可行的。只是原理是什麼呢？

答案揭曉：把應用程式註冊在電腦裡，做出一個屬於它的 protocol。

Windows版本（我實際操作的版本）

其實過程滿簡單的，以隨意呼叫一個 .exe 檔而言，我寫了一個 .reg 檔如下


Windows Registry Editor Version 5.00


[HKEY_CLASSES_ROOT\ATIS]
@="URL:ATIS Protocol"	    # 註冊 protocol "ATIS"
"URL Protocol"=""


[HKEY_CLASSES_ROOT\ATIS\DefaultIcon]
@="D:\\ATIS\\bin\\Debug\\ATIS.exe"
# 是 ATIS.exe 的路徑

[HKEY_CLASSES_ROOT\ATIS\shell]
@=""


[HKEY_CLASSES_ROOT\ATIS\shell\open]
@=""


[HKEY_CLASSES_ROOT\ATIS\shell\open\command]
@="D:\\ATIS\\bin\\Debug\\ATIS.exe \"%1\""
# 這裡會打開 shell 執行命令，"%1" 可以將URL本身傳為 ATIS.exe 的外部參數

Windows style，用 regedit 開啓這個 .reg 檔，馬上會有一個警告問你要不要執行把它登錄為機碼，自己寫的東西不用太害怕，點 是 就好。

登錄進去以後在 HKEY_CLASSES_ROOT 就會看到自己寫的 .reg 檔成功登錄的資料夾結構，就是 .reg 檔中括號的內容，完全可以對應。

這時候隨意寫一個 HTML 網頁起來做測試

<html>
<head></head>
<body>
	<a href="ATIS://WhateverYouWriteThisContent">Test !</a>
</body>
</html>

點擊以後，就可以叫起 ATIS.exe 檔了。這麼做的好處就是可以

  1. 不需要使用 IE 的 ActiveX
  2. 所有瀏覽器都適用，只要他們有安裝該應用程式並登錄為機碼

Reference

因為目前的環境都是 Windows 下的，所以我參考了這一篇MSDN文件：Registering an Application to a URI Scheme

另外我想 OSX 應該也有相對應的做法，於是也找到了這篇文章：OS X URL handler to open links to local files

By the way， HKEY_CLASSES_ROOT\ATIS\shell\open\command 不止開檔案而已，是可以直接在 shell 下 command 的…
